Activate 1.0.1 - Binding to nodes with length

Back to changelog

In Activate 1.0.0, the check for whether activate's nodes argument was a single Node or multiple was based on the presence or absence of a truthy length property. However, this resulted in false positives when the passed Node was a <select> element, as they have a length property.

This was then causing an error because the Node did not contain a forEach method.

This check now runs nodes instanceof Node first, then if that check fails it checks for both a truthy length property and for the presence of forEach.

Breaking changes

  • None

Non-breaking changes

  • Treat Node with truthy length properties as a single Node
